Ключевое открытие

Приложение работает в браузере, но использует Python внутри браузера через технологию Pyodide. Вся логика генерации IFC выполняется на вашем компьютере, без отправки данных на сервер.

JavaScript (main.js)
async function downloadIFCFile() {
    // 1. Получаем текущие параметры
    const params = form.getParams();
    // 2. Перегенерируем болт
    await generateBolt(params);
    // 3. Получаем IFC-данные из моста
    const ifcData = bridge.getIFCData();
    // 4. Скачиваем файл
    const filename = generateFilename(params);
    UI.downloadFile(ifcData, filename);
}

Что происходит на каждом шаге:

form.getParams() — Собирает выбранные вами параметры: тип болта, диаметр, длину, материал
generateBolt(params) — Отправляет параметры в Python через «мост», получает обратно 3D-геометрию
bridge.getIFCData() — Забирает готовый IFC-файл как текст
UI.downloadFile() — Создаёт файл и предлагает браузеру сохранить его

Как части общаются: JS ↔ Python

Магия Pyodide и моста между мирами

JavaScript и Python — два разных языка, которые работают в разных «вселенных». Pyodide создаёт мост между ними, но для общения нужен «переводчик» — ifcBridge.js.

↓
2
Конвертация JS-объекта в JSON
↓
3

↓
4

↓
5
Python возвращает результат (Proxy)
↓
6
Конвертация Proxy → JS-объект
⚠️
Тонкий момент: Proxy-объекты

Когда Python возвращает данные в JavaScript, они приходят в специальной «обёртке» — Proxy. Мост автоматически конвертирует их обратно в обычные JS-объекты через метод toJs().

ifcBridge.js — вызов Python
async generateBolt(params, exportSettings) {
    // 1. Конвертация JS → Python-синтаксис
    const paramsJson = JSON.stringify(params)
        .replace(/false/g, 'False')
        .replace(/true/g, 'True');
    // 2. Создание Python-кода
    const result = await this.pyodide.runPythonAsync(`
        from instance_factory import generate_bolt_assembly
        params = ${paramsJson}
        ifc_str, mesh_data = generate_bolt_assembly(params)
        (ifc_str, mesh_data)
    `);
    // 3. Конвертация Proxy → JS
    const meshData = this.convertPyodideObject(result[1]);
    return { ifcData: result[0], meshData, status: 'success' };
}

Три ключевых шага:

JSON.stringify + replace — Превращает JS-объект в строку, заменяя true/false на True/False (Python-синтаксис)
pyodide.runPythonAsync() — Выполняет Python-код внутри браузера, возвращает результат
convertPyodideObject() — Рекурсивно обходит Proxy, превращает его в обычный JS-объект

Модуль 4

IFC: язык BIM-моделей

Как код становится строительной моделью

IFC — это не просто 3D-модель. Это структурированное описание здания со всеми компонентами, материалами и связями между ними. Файл .ifc содержит сотни сущностей вроде IfcMechanicalFastener (болт) и IfcExtrudedAreaSolid (геометрия).

🏗️

IfcProject

Корневой объект. Представляет весь строительный проект. Содержит единицы измерения, систему координат, контекст.

🏢

IfcBuildingStorey

Этаж здания. Все элементы (включая болты) «прикреплены» к этажу через IfcRelContainedInSpatialStructure.

🔩

IfcMechanicalFastener

Крепёжный элемент (болт, гайка, шайба). Имеет тип, материал, позицию в пространстве.

📐

IfcExtrudedAreaSolid

Геометрия: 2D-профиль, «выдавленный» вдоль оси. Так создаются гайки, шайбы, плиты.

🌀

IfcSweptDiskSolid

Геометрия: кривая, «протянутая» с круглым сечением. Так создаются изогнутые шпильки.

🔗

IfcRelAggregates

Связь «состоит из». Assembly (сборка) → Components (компоненты: шпилька, гайки, шайба).

💡
Иерархия IFC-документа

IfcProject → IfcBuildingStorey → IfcMechanicalFastener (сборка) → IfcRelAggregates → компоненты (шпилька, гайки, шайба). Каждый уровень добавляет детали: от общего контекста к конкретной геометрии.

Вы открываете IFC-файл в текстовом редакторе и видите строку: #1234=IFCMECHANICALFASTENER(...,NAME='bolt_1.1_M20x800',...). Что это означает?

B

D

ГОСТ в коде: данные и геометрия

Как нормативный документ становится программой

ГОСТ 24379.1-2012 — это 50+ страниц таблиц, формул и чертежей. Приложение превращает их в Python-данные: словари с размерами, функции для расчёта массы, валидацию параметров.

📊

Уровень 1: CSV

data/dim.csv — сырые данные из ГОСТ. Одна строка = один диаметр+длина. Пример: 20_800: [800, 60, 20, 20, 100, 2.31, ...]

🐍

Уровень 2: Python-модули

bolt_dimensions.py — преобразует CSV в словари. get_bolt_dimensions(20, 800) возвращает {L: 800, l: 60, R: 20, ...}

🔧

Уровень 3: Геометрия

geometry_builder.py — использует данные для построения 3D-кривых. Тип 1.1: дуга радиусом R, тип 5: прямая линия

Тип 1.1: шпилька с крюком (загиб 90°). Тип 1.2: шпилька с двойным загибом. Тип 5: прямая шпилька. geometry_builder.py строит кривые через IfcIndexedPolyCurve с IfcArcIndex для дуг.

3D в браузере: от IFC к Three.js

Конвертация BIM-модели в интерактивную визуализацию

IFC-файл содержит точную параметрическую геометрию, но браузер не умеет её рисовать напрямую. geometry_converter.py использует ifcopenshell.geom, чтобы получить вершины и грани, а viewer.js создаёт из них mesh для Three.js.

⚠️
Проблема систем координат

IFC использует Z-up (вертикаль — ось Z), Three.js — Y-up (вертикаль — ось Y). viewer.js трансформирует вершины: (x, y, z) → (x, -z, -y). Без этого болт лежал бы «на боку».

viewer.js — трансформация вершин
transformVerticesForThreeJS(vertices) {
    const transformed = [];
    for (let i = 0; i < vertices.length; i += 3) {
        // IFC: Z-вертикаль, метры
        // Three.js: Y-вертикаль, миллиметры
        transformed.push(
            vertices[i] * 1000,       // x (м → мм)
            -vertices[i + 2] * 1000,  // -z → y
            -vertices[i + 1] * 1000  // -y → z
        );
    }
    return transformed;
}

Три преобразования:

* 1000 — Конвертация метров (IFC) в миллиметры (Three.js). IFC хранит в метрах, даже если указаны мм
-z → y — Ось Z из IFC становится осью Y в Three.js с инверсией (чтобы вид сверху совпадал)
-y → z — Ось Y из IFC становится осью Z в Three.js с инверсией для правильной ориентации
После генерации болта вы видите в консоли: вершины [0, 0, 0], [0, 0, -0.8], ... (в метрах). Какими они станут после трансформации для Three.js?
A
[0, 0, 0], [0, 0, -800] — просто умножение на 1000
B
[0, 0, 0], [0, 800, 0] — Z становится Y с инверсией
C
[0, 0, 0], [-800, 0, 0] — Z становится X
D
[0, -800, 0], [0, 0, 0] — инверсия Y

Ключевые принципы архитектуры
