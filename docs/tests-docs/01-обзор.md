# Обзор тестирования

## Назначение

Проект использует Test-Driven Development (TDD) подход для разработки Python и JavaScript модулей.

## Уровни тестирования

| Уровень       | Инструмент       | Описание                                         |
| ------------- | ---------------- | ------------------------------------------------ |
| Unit-тесты    | pytest           | Python модули (**305 тестов**)                   |
| Unit-тесты    | Jest             | JavaScript модули (**99 тестов**)                |
| Валидация IFC | Behave (Gherkin) | Проверка по правилам buildingSMART (100+ правил) |

## Статистика тестов

### Python тесты

```
======================== 305 passed, 2 skipped in 13.36s ========================
```

- **305 тестов пройдено**
- **2 теста пропущено** (требуют ifcopenshell.geom)
- **0 тестов упало**
- **Покрытие: 89%**

### JavaScript тесты

```
Test Suites: 6 passed, 6 total
Tests:       99 passed, 99 total
```

- **99 тестов пройдено**
- **0 тестов упало**
- **Покрытие: 100%**

> **Примечание:** Для запуска JavaScript тестов установите зависимости: `npm install`

## Структура тестирования

```
tests/
├── Python тесты (pytest)
├── Gherkin валидация (behave)
└── Фикстуры и Mock-классы

js/tests/
└── JavaScript тесты (Jest)
```
