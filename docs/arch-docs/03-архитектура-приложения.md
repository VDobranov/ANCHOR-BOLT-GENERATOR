# Архитектура приложения

## Общая структура

Приложение построено по клиент-серверной архитектуре, где вся обработка выполняется в браузере клиента.

```
┌─────────────────────────────────────────────────────────────┐
│                     Браузер (Client)                        │
├─────────────────────────────────────────────────────────────┤
│  JavaScript слой                                            │
├─────────────────────────────────────────────────────────────┤
│  Python слой (Pyodide WebAssembly)                          │
└─────────────────────────────────────────────────────────────┘
```

## JavaScript слой

### Ответственность

- Отображение пользовательского интерфейса
- Валидация введённых параметров
- Управление 3D-сценой (Three.js)
- Координация работы Python-модулей

### Основные модули

| Модуль         | Ответственность               |
| -------------- | ----------------------------- |
| `ui.js`        | Управление формами, статусами |
| `form.js`      | Обработка параметров болта    |
| `viewer.js`    | 3D-визуализация (Three.js)    |
| `ifcBridge.js` | Мост JS ↔ Python (Pyodide)    |
| `main.js`      | Оркестрация приложения        |

## Python слой

### Ответственность

- Создание IFC-документа
- Построение параметрической геометрии
- Генерация IFC-сущностей
- Конвертация IFC → mesh

### Основные модули

| Модуль                  | Ответственность                  |
| ----------------------- | -------------------------------- |
| `main.py`               | Точка входа, менеджер документов |
| `container.py`          | DI-контейнер                     |
| `type_factory.py`       | Кэширование типов болтов         |
| `instance_factory.py`   | Создание экземпляров             |
| `geometry_builder.py`   | Построение геометрии IFC         |
| `geometry_converter.py` | Конвертация IFC → mesh           |
| `material_manager.py`   | Управление материалами           |

## Взаимодействие слоёв

## Жизненный цикл приложения

### Этап 1: Инициализация

1. Загрузка Pyodide (~80 МБ)
2. Установка зависимостей (micropip)
3. Загрузка Python-модулей в FS
4. Создание базовой IFC-структуры

### Этап 2: Генерация болта

1. Ввод параметров пользователем
2. Валидация (JS + Python)
3. Построение геометрии
4. Конвертация в mesh
5. Обновление 3D-сцены

### Этап 3: Экспорт

1. Формирование IFC-документа
2. Сериализация в строку
3. Скачивание файла браузером
