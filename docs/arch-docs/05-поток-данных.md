# Поток данных

## Обзор

Поток данных описывает последовательность преобразования пользовательского ввода в 3D-модель и IFC-файл.

## Этапы потока

```
┌──────────┐     ┌───────────┐     ┌─────────────┐
│  Ввод    │ ──► │ Валидация │ ──► │ TypeFactory │
│параметров│     │ (JS+Python)│     │ (кэш типов) │
└──────────┘     └───────────┘     └─────────────┘
                                          │
                                          ▼
┌──────────┐     ┌───────────┐     ┌─────────────┐
│  Viewer  │ ◄── │ Geometry  │ ◄── │  Instance   │
│(обновление│     │ Converter │     │   Factory   │
│   сцены) │     │(IFC→mesh) │     │(экземпляры) │
└──────────┘     └───────────┘     └─────────────┘
```

## 1. Ввод параметров

### Пользовательский ввод

Пользователь выбирает параметры в форме:

| Параметр  | Пример  | Источник              |
| --------- | ------- | --------------------- |
| Тип болта | "1.1"   | Select (статичный)    |
| Диаметр   | 20      | Select (динамический) |
| Длина     | 800     | Select (динамический) |
| Материал  | "09Г2С" | Select (статичный)    |

### Динамическое обновление

При изменении типа/диаметра:

1. Фильтрация доступных длин
2. Обновление списка длин
3. Пересчёт массы

## 2. Валидация

### JavaScript валидация (клиент)

Быстрая проверка перед отправкой в Python:

- Проверка наличия всех параметров
- Проверка типов данных
- Проверка диапазонов значений

### Python валидация (Pyodide)

Глубокая проверка по ГОСТ:

- `validate_parameters()` — проверка комбинации параметров
- Проверка доступности длины для типа/диаметра
- Проверка материала

## 3. TypeFactory

### Назначение

Кэширование типов болтов для оптимизации размера IFC.

### Процесс

```
Запрос типа → Проверка кэша → Создание нового → Кэширование
```

### Ключ кэша

```python
key = (bolt_type, execution, diameter, length, material)
```

### RepresentationMaps

Для каждого типа создаётся:

- `IfcMechanicalFastenerType`
- `IfcRepresentationMap` с геометрией
- `IfcShapeRepresentation` (Body)

## 4. InstanceFactory

### Назначение

Создание экземпляров компонентов болта.

### Процесс

1. Получение типа из TypeFactory
2. Создание `IfcMechanicalFastener` для каждого компонента
3. Позиционирование через `IfcLocalPlacement`
4. Привязка материала

### Компоненты сборки

| Компонент | Позиция     | Материал |
| --------- | ----------- | -------- |
| Шпилька   | (0, 0, 0)   | Из типа  |
| Гайка #1  | (0, 0, L)   | Из типа  |
| Гайка #2  | (0, 0, L+h) | Из типа  |
| Шайба #1  | (0, 0, L)   | Из типа  |

## 5. GeometryBuilder

### Назначение

Построение параметрической IFC-геометрии.

### Типы геометрии

| Компонент | Тип геометрии                               |
| --------- | ------------------------------------------- |
| Шпилька   | IfcSweptDiskSolid                           |
| Гайка     | IfcExtrudedAreaSolid (шестиугольник)        |
| Шайба     | IfcExtrudedAreaSolid (кольцо)               |
| Плита     | IfcExtrudedAreaSolid (квадрат с отверстием) |

### Построение шпильки

1. Создание составной кривой (ось)
2. Определение радиуса (d/2)
3. Создание IfcSweptDiskSolid

## 6. GeometryConverter

### Назначение

Конвертация IFC-геометрии в mesh Three.js.

### Процесс

```python
shape = ifcopenshell.geom.create_shape(ifc_entity)
vertices = shape.geometry.verts
faces = shape.geometry.faces
```

### Преобразование

- Вершины: массив [x1,y1,z1, x2,y2,z2, ...]
- Грани: индексы [0,1,2, 0,2,3, ...]
- Нормали: вычисляются автоматически

## 7. Viewer (обновление сцены)

### Three.js mesh

```javascript
const geometry = new THREE.BufferGeometry();
geometry.setAttribute('position', new THREE.Float32BufferAttribute(vertices, 3));
geometry.setIndex(faces);
const material = new THREE.MeshStandardMaterial({ color });
const mesh = new THREE.Mesh(geometry, material);
```

### Обновление сцены

1. Очистка предыдущей геометрии
2. Добавление новых mesh-объектов
3. Центрирование камеры
4. Обновление освещения
