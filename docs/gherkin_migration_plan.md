### Пример step definition (Python)

```python
# features/steps/thens/existence.py

from behave import then
from step_helpers import file_handling

@then('there must be {min_count} instance(s) of {entity_type}')
def step_impl(context, min_count, entity_type):
    min_count = int(min_count)
    instances = file_handling.get_entities_by_type(context, entity_type)
    
    assert len(instances) >= min_count, \
        f"Expected at least {min_count} {entity_type}, found {len(instances)}"
```

## 🔄 Процесс обновления правил

### Текущий (pytest)
```
1. buildingSMART обновляет правила
2. Мы узнаём об изменениях (мониторинг)
3. Вручную обновляем тесты
4. Запускаем тесты
5. Коммитим изменения
```

**Риски:** Человеческий фактор, отставание от стандарта

### Целевой (Gherkin + submodule)
```
1. buildingSMART обновляет правила
2. git submodule update
3. Запускаем валидацию
4. Фиксим breaking changes (если есть)
```

**Преимущества:** Автоматические обновления, меньше ручной работы
