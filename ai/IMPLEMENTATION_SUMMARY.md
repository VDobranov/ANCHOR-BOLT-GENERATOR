### Python модули ✅
1. **main.py** - Инициализация IFC4 ADD2 TC1 документа
   - Создание Project/Site/Building/Storey структуры
   - Обработка ошибок Pyodide
   - GUID генерация через ifc.guid.new()

2. **gost_data.py** - Справочники ГОСТ 24379.1-2012
   - Все 4 типа болтов (1.1, 1.2, 2.1, 5)
   - 8 диаметров (М12-М48) по DIM.py (BlenderBIM)
   - Полные таблицы длин по типам и исполнениям (93 комбинации)
   - 3 материала с характеристиками
   - Валидация параметров

3. **geometry_builder.py** - Геометрические примитивы
   - Создание линий и дуг
   - IfcIndexedPolyCurve для типа 1.1 (подход BlenderBIM)
   - IfcCompositeCurve для типов 1.2, 2.1, 5
   - Профили для гаек/шайб (IfcArbitraryProfileDefWithVoids)
   - SweptDiskSolid для шпилек

4. **type_factory.py** - Кэширование типов
   - Уникальное кэширование по (type, diameter, length, exec, material)
   - Создание типов: Stud, Nut, Washer, Assembly
   - Оптимизация размера IFC файла

5. **instance_factory.py** - Создание инстансов и сборок
   - Автоматический расчёт позиций компонентов
   - Создание отношений (RelDefinesByType, RelAggregates, RelConnectsElements)
   - Генерация mesh-данных через ifcopenshell.geom

6. **geometry_converter.py** - Конвертация IFC → Three.js mesh
   - Использование ifcopenshell.geom.create_shape()
   - Извлечение вершин, индексов, нормалей
   - Конвертация метров в миллиметры

7. **ifc_generator.py** - Экспорт IFC
   - Сохранение в SPF формат
   - Валидация документа
   - Получение информации о структуре

8. **utils.py** - Общие утилиты
   - Централизованный импорт ifcopenshell (get_ifcopenshell)
   - Проверка доступности ifcopenshell

## 📐 Архитектурные решения

### Кэширование типов
```
Key: ('stud', '1.1', 1, 20, 800, '09Г2С')
→ Один раз созданный тип переиспользуется
→ Оптимизирует IFC файл и производительность
```

### Двухэтапная инициализация
1. **При загрузке** (one-time):
   - Pyodide runtime (~80 МБ)
   - Установка зависимостей: typing_extensions → numpy → ifcopenshell
   - IFC базовая структура
   - Кэширование типов

2. **При генерации** (per-request):
   - Валидация параметров
   - Создание инстансов
   - Конвертация IFC геометрии в mesh через ifcopenshell.geom
   - Рендеринг 3D и IFC

### IFC иерархия
```
IfcProject
└── IfcSite
    └── IfcBuilding
        └── IfcBuildingStorey
            └── IfcMechanicalFastener (Assembly: ANCHORBOLT)
                ├── IfcMechanicalFastener (Stud: USERDEFINED)
                ├── IfcMechanicalFastener (Washer #1: USERDEFINED)
                ├── IfcMechanicalFastener (Nut #1: USERDEFINED)
                └── IfcMechanicalFastener (Nut #2: USERDEFINED)
```

**Состав сборки определяется автоматически:**
- Типы 1.1, 1.2, 5: шпилька + шайба + 2 гайки
- Тип 2.1: шпилька + шайба + 2 верхних гайки + 2 нижних гайки

## 💡 Ключевые особенности реализации

1. **Полная валидация ГОСТ** - все параметры проверяются против стандартов
2. **Умное кэширование** - типы создаются один раз и переиспользуются
3. **3D визуализация** - интерактивный просмотр с Three.js (ортографическая камера)
4. **IFC экспорт** - стандартный формат для BIM систем (IFC4 ADD2 TC1)
5. **Браузерное выполнение** - нет необходимости в серверной части
6. **Горячее обновление** - изменения параметров мгновенно обновляют вид
7. **Единый источник геометрии** - IFC геометрия строится один раз, конвертируется через ifcopenshell.geom
8. **Централизованный импорт** - все модули используют utils.py для импорта ifcopenshell
